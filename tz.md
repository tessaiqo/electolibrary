## Разработка SPA-приложения на Vue 3 с сервером на Python

Данное задание предназначено для отработки основных возможностей Vue 3, включая работу с данными, формами, компонентами, слотами, маршрутизацией, а также взаимодействие с сервером.
В рамках задания вы создаёте полноценное SPA-приложение и серверную часть на Python с запуском в Docker.

---

# 🎯 Цели задания

1. Освоить фундаментальные возможности Vue 3:

   * привязка данных, события, computed, watch;
   * формы и модификаторы ввода;
   * условный рендеринг;
   * вывод массивов, сортировка, фильтрация;
   * компоненты, props, события, слоты;
   * маршрутизация Vue Router;
   * работа с refs и жизненным циклом.
2. Научиться взаимодействовать с сервером (REST API).
3. Освоить базовую серверную разработку на Python (FastAPI или Flask).
4. Освоить контейнеризацию приложения (Docker).
5. Научиться структурировать отчёт по проекту.

---

# 📝 Суть задания

Разработать мини-приложение «Электронная библиотека» (**ElectoLibrary**) с клиентской частью на Vue 3 и серверной частью на Python.

Приложение должно уметь:

* отображать список книг,
* создавать новую книгу в коллекции,
* редактировать поля книги (автор, издательство, год издания и т.д.),
* удалять книгу из коллекции,
* фильтровать и сортировать книги,
* работать со страницами (маршруты Vue Router),
* использовать компоненты и слоты,
* сохранять данные на сервере (в SQLite или PostgreSQL),
* запускаться через Docker.

---

# 🚀 Функциональные требования

## Клиентская часть (Vue 3)

### 1. Главная страница `/`

* Приветственный экран или Dashboard.
* Ссылка/кнопка «Электронный каталог».

### 2. Страница списка книг `/books`

Должны быть реализованы:

* Вывод книг (`v-for`)
* Фильтрация:

  * по статусу (все / в наличии )
* Сортировка:

  * по дате добавления
  * по алфавиту (поле: название)
* Условный рендеринг пустого списка (`v-if`)
* Компонент `BookItem.vue`, принимающий **props**
* Передача событий от дочернего компонента к родительскому
* Кнопки:

  * "Редактировать"
  * "Удалить"
  * "Изменить статус"

### 3. Страница создания книги `/books/new`

Форма с такими полями (основываться на ГОСТ 2018 года):

* Заголовок (input)
* Автор (input)
* Описание (textarea)
* Обложка (файл jpg)
* Издательство (select)
* Категория (radio): (aka «возрастной рейтинг»)


Требования:

* `v-model` с модификаторами
* Валидация
* Отправка данных на сервер

### 4. Страница редактирования `/books/:id/edit`

То же, что и создание, но поля заполнены данными.

### 5. Компоненты

Обязательные компоненты:

* `BookList.vue` — список книг
* `BookItem.vue` — отображение одной книги
* `BookForm.vue` — форма создания/редактирования
* `AppHeader.vue` — верхнее меню
* `AppFooter.vue` — нижняя панель
* Компонент с **слотом** (например, LayoutCard.vue)

### 6. Слоты

Использовать:

* Обычный слот
* Именованный слот
* Слот с ограниченной областью видимости

### 7. Маршрутизация

Должно быть реализовано:

* маршруты
* вложенные маршруты
* именованные маршруты
* программная навигация
* страница 404

# 🖥️ Серверная часть (Python)

## Требования

Можно выбрать один из фреймворков:

* **FastAPI** 

Сервер должен содержать:

### 1. CRUD-методы

```
GET /api/books
GET /api/boooks/{id}
POST /api/books
PUT /api/books/{id}
DELETE /api/books/{id}
```

### 2. Хранение данных

Варианты:

* **SQLite** — файл `tasks.db`
* **PostgreSQL** — для выгрузки/загрузки использовать pgdump / pg_restore

### 3. Логика обновления файла/БД

При каждом запросе данные должны записываться, чтобы студент/преподаватель мог посмотреть содержимое вручную.

### 4. Запуск через Dockerfile / docker compose
